Apple keys and certificates

2019-02-21
ios

Contents

  • 1. Apple Development
  • 1. Mobile provision keys (.mobileprovision file)

1. Apple Development

This file is used to added to Xcode, then developer can run app on device.

1. Mobile provision keys (.mobileprovision file)

It is created from Apple Developer Center. Go to Profiles -> Select plus button -> Select iOS App Development (Create a provisioning profile…) -> Select App ID which created from a -> In Select Certificate and Select Devices, select all -> Download .mobileprovision file

2. Certificate Signing Request (.certSigningRequest file)

It is created from developer’s laptop. Step 1: Open Keychain Step 2: Follow like photo below Step 3: Put email address and select Saved to disk instead of Emailed to the CA Step 4 Tick on Let me specify key pair information Step 5: Next Next -> until finish

3. App Store Connect

Apple change the name iTune Connect to App Store Connect since 2018.

4. Provisioning profile

The file that store bundle id, export type (app-store or ad-hoc), team information. It togethers with provision profile in order to upload the app to store. It is valid one year after generation.

Creation
Step 1: Go to Developer Portal, login
Step 2: Go to Identifiers tab, create new Identifiers
Step 3: Go to Profiles tab, create new Profiles with that identifier
Step 4: Download .mobileprovisioning file

5. Code signing certificate (.p12)

The file that store developer information and team developer information. It togethers with provisioning profile in order to upload the app to store. It is valid one year after generation

Creation
Step 1: Create CSR file (on local)
Step 2: Go to Certificates tab, create new Certificates by CSR file.
Step 3: Download Certificates (.cer file), then double click on .cer in order to install this cerficate on local.
Step 4: Open Keychain access, export that certificate to .p12 file. (can set password for .p12 file)
Read the link here: https://stackoverflow.com/a/28962937/8120939
