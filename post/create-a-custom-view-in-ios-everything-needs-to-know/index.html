Create a custom view in iOS: Everything needs to know

2018-08-19
ios

Contents

    • 3. Create Custom UITableViewCell
    • 4. Create Custom UICollectionViewCell
    • 5. Create Custom UIViewController
Share on:

1. Introduction

In iOS programming, we usually need to create custom views. We create them not only for reused, in some cases it also help us move a lot of code to other class. In this article, I will share you guys everything we need to know when we create a custom view. What is difference between when we create with nib file and without it. OK, let go…

2. Create custom view without Nib.

There are 2 cases:
- Put view into storyboard
+ It will call required init?(coder aDecoder: NSCoder), then it calls awakeFromNib()
+ It will not call override init(frame: CGRect)
- Create view programmingly
+ It will only call override init(frame: CGRect) if you init with frame, it will only call init(coder: …) if you create view by init() + It does not call any other functions, even awakeFromNib()
Note: - In case we have outlet, outlet can be used in awakeFromNib function. We can use outlet before or after super.awakeFromNib(), but outlets are nil in all of function call before awakeFromNib(). See more about UIView lifecycle here.
- We cannot create custom view with nib. Nib must go with UIViewController or UITableViewCell, UICollectionViewCell.

3. Create Custom UITableViewCell

There are 2 cases:
- Register cell class
+ Only override init(style: UITableViewCell.CellStyle, reuseIdentifier: String?) is called
+ required init?(coder aDecoder: NSCoder) and awakeFromNib not be called
- Register cell from Nib
+ It call required init?(coder aDecoder: NSCoder) then it call awakeFromNib
+ override init(style: UITableViewCell.CellStyle, reuseIdentifier: String?) not be called

4. Create Custom UICollectionViewCell

There are 2 cases:
- Register cell class
+ Only override init(frame: CGRect) is called
+ required init?(coder aDecoder: NSCoder) and awakeFromNib not be called
- Register cell from Nib
+ It call required init?(coder aDecoder: NSCoder) then it call awakeFromNib
+ override init(frame: CGRect) not be called
Note: Custom UICollectionViewCell is exactly the same with Custom UITableViewCell

5. Create Custom UIViewController

Rule:
- Do not write init() function.
- Use loadView() and viewDidLoad() functions instead.

class ZoomImageVC: UIViewController {
    lazy var imageView: UIImageView = {...}()

    override func loadView() {
        super.loadView()
        // Do something here, like setup UI
    }

    override func viewDidLoad() {
        super.viewDidLoad()
        // Do viewDidLoad here, like assign image for imageView
    }
}
